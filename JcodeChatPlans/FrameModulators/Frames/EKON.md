## Phase Structure: The 6 Collapse Operations

Unlike IDEVA's evolutionary phases, ΣΚΟΠ's phases are **collapse operations** — each one reduces the possibility field. They are not sequential; they are applied in order of available constraint density. You apply whichever operation gives the largest field reduction first.

***

### Collapse 1 — Field Definition (What You're Actually Looking For)

The single most common finding failure: searching for the wrong target.

- **Compress the target description** until it cannot be confused with anything else. If two different things match your description, your description is still too loose
- **Extract the Eigenstate (D5):** What is the *one property* only this target has? Not what it looks like — what it *is*. A face has millions of features; its Eigenstate is the unique ratio between specific feature pairs
- **Map the anti-target:** What is definitively NOT what you're looking for? Anti-target definition often collapses the field faster than target definition
- **Output:** `TargetSignature = {eigenstate, anti-target, minimum_distinguishing_features}`

**Applied example (your context):** Looking for "a market gap in AI tooling" — wrong. Compress: "a workflow where enterprise teams currently use >3 manual steps that could be replaced by a single local inference call, generating >\$10k/year in time savings per team." Now the search space has hard edges.

***

### Collapse 2 — Space Compression (Eliminate Before You Search)

Never search a space you can eliminate. This is the highest-leverage operation and the most underused.

- **Apply the Constraint Engine:** For each known property of the target, what does it *logically exclude*? Build the exclusion set before entering the search space
- **Shadow mapping (D4 — Resonance):** What does the target leave behind or attract? Targets leave shadows — behavioral traces, correlations, second-order effects. Find the shadow first, then locate the source
- **Fractal pruning:** If the target is not in the first 20% of the most likely locations, the structure of the remaining 80% can be recursively pruned using the same constraint set
- **Output:** `PrunedSpace = original_space × (1 - exclusion_ratio)` — the smaller this number before active search begins, the more advanced your finding is

**The anti-pattern:** Starting search before running Collapse 2. Most search tools (Google included) give you D1+D2 but force you to run Collapse 2 manually. ΣΚΟΠ makes it the *first* operation.

***

### Collapse 3 — Signal Amplification (Make the Target Loud)

Instead of searching harder, make the target more detectable.

- **Resonance injection (D4):** Introduce a signal into the search space that only the target would respond to. In biology this is a ligand; in markets this is a pricing signal; in data this is a probe query; in social finding this is a specific question only the right person knows the answer to
- **Contrast maximization:** Surround the search space with anti-signal so the target stands out maximally. The target becomes visible not by illuminating it but by darkening everything around it
- **Temporal compression:** If the target changes over time, find the *rate of change* rather than the current state. Anomalies in velocity are visible before anomalies in position
- **Output:** `AmplifiedSignature` — the target's signal after environmental contrast maximization

**Quantum leap here:** Most finding is passive (scan and detect). Signal amplification makes finding *active* — you alter the environment to make the target reveal itself. This is the difference between a metal detector and a magnet.

***

### Collapse 4 — Multi-Dimensional Triangulation

A target invisible in one dimension is locatable from the intersection of multiple partial signals.

- **Run parallel searches across all 5 dimensions simultaneously** — even if each dimension gives only a weak signal, their intersection is exponentially stronger
- **Contradiction triangulation:** If two information sources give contradictory signals about the target's location, the contradiction itself is a high-precision locator — the target is at the boundary where both signals are partially true
- **Temporal triangulation:** Where was the target? Where is it now? Extrapolate. The trajectory is often more locating than the current position
- **Absence mapping:** If the target should appear in a location and doesn't, its absence is as locating as its presence — you learn the shape of the void it leaves
- **Output:** `IntersectionPoint` — the region where all partial signals converge; your highest-confidence current target location

***

### Collapse 5 — Eigenstate Verification (Is This Actually It?)

The finding is not complete until verification runs against the Eigenstate defined in Collapse 1.

- **Apply the minimum distinguishing test:** Does what you found have the *one property* only the target has? If yes, confirmed. If no, it's a false positive — return to Collapse 2 with the false positive as an additional anti-target
- **Adversarial check:** Run the strongest possible argument that what you found is NOT the target. If the argument fails, confidence is high. If the argument succeeds, you have a false positive and new constraint information
- **Completeness check:** Is this the *only* instance, or are there more? If the target is a pattern rather than a singleton, Collapse 5 triggers a new field definition for the next instance
- **Output:** `{confirmed: bool, confidence: 0-1, false_positive_constraints: [], completeness: singleton|pattern}`

***

### Collapse 6 — Eigenstate Encoding \& Search Map Update

The self-referential operation that makes ΣΚΟΠ compound over time.

- **Encode what you found** into the D5 eigenstate library — each confirmed find enriches the library of distinguishing signatures
- **Update the exclusion set** — what you found also tells you what else doesn't need to be found in the same space
- **Propagate the shadow map** — the target's resonance pattern (D4) is now known; future searches for related targets can use this as a probe signal
- **Compress the search process itself** — what was the critical constraint that collapsed the field fastest? That constraint goes into the `FastPath` register for similar future searches
- **Output:** `SearchMapDelta` — the update to the global finding model that makes every future search faster

This is the self-reference loop. Each search teaches the system how to search. After N searches, the system doesn't search the space anymore — it knows where things are *before it looks*.
